## Overview

CUCEIUbicate uses a custom secure hashing mechanism for password management. All passwords are hashed using a multi-pass algorithm with a static salt and timestamp-based entropy.

## Password Security

The authentication system implements:

- **Multi-pass hashing**: Three separate hash passes for enhanced security
- **Static salt**: `CUCEI_UBICATE_2024_PRODUCTION_SECURE_SALT_V2`
- **Timestamp entropy**: Adds uniqueness to each hash
- **String normalization**: Prevents encoding-related authentication issues

### Hash Format

```
$secure$[hash1]$[hash2]$[hash3]$[timestamp]
```

Example:
```
$secure$0a1b2c3d$4e5f6g7h$9i0j1k$abc123
```

### Validation Rules

#### Email Validation
- Must match pattern: `^[^\s@]+@[^\s@]+\.[^\s@]+$`
- Must be unique in the system
- Automatically converted to lowercase

#### Username Validation
- Minimum length: 3 characters
- Maximum length: 30 characters
- Allowed characters: `a-z`, `A-Z`, `0-9`, `.`, `-`, `_`
- Must be unique in the system
- Automatically converted to lowercase

#### Password Validation
- Minimum length: 6 characters
- Maximum length: 50 characters (before hashing)
- Stored as secure hash (not reversible)

## User Login

Authenticate a user with email/username and password.

### Authentication Method

The login system accepts either:
- Email address
- Username

Both are normalized to lowercase for comparison.

### Login Process

1. **Input normalization**: Remove diacritics and convert to lowercase
2. **User lookup**: Search by email first, then by username
3. **Password verification**: Compare using secure hash algorithm
4. **Session data**: Return complete user profile on success

### Implementation

<CodeGroup>

```typescript TypeScript
import { login } from './api';

const result = await login(
  'juan.perez@alumnos.udg.mx',  // email or username
  'MySecurePass123'              // password
);

if (result && result.isMatch) {
  console.log('Login successful!');
  console.log('User data:', result.userData);
  
  // Access user information
  const { id, email, username, name, student_code } = result.userData;
} else {
  console.error('Invalid credentials');
}
```

```javascript JavaScript
import { login } from './api';

const result = await login('juanp', 'MySecurePass123');
```

## Password Hashing

Understanding the secure hash implementation:

### Hash Generation

```typescript
function secureHash(password: string): string {
  const staticSalt = 'CUCEI_UBICATE_2024_PRODUCTION_SECURE_SALT_V2';
  const timestamp = Date.now().toString(36).slice(-6);
  const combined = password + staticSalt + timestamp;
  
  // Three separate hash passes with different algorithms
  let hash1 = multiPassHash(combined, 5);  // Shift left 5
  let hash2 = multiPassHash(combined, 3);  // Shift left 3
  let hash3 = multiPassHash(password + staticSalt, 7);  // Shift left 7
  
  return `$secure$${hash1}$${hash2}$${hash3}$${timestamp}`;
}
```

### Hash Verification

Verification recreates the hash using the stored timestamp:

```typescript
function secureCompare(password: string, hashedPassword: string): boolean {
  // Extract timestamp from stored hash
  const parts = hashedPassword.split('$');
```

## Best Practices

<AccordionGroup>
  <Accordion title="Store user session data securely">
    After successful login, store the user data in secure storage:
    - Use encrypted AsyncStorage on React Native
    - Never store passwords in plain text
    - Clear session data on logout
  </Accordion>
  
  <Accordion title="Handle password validation on client side">
    Validate password requirements before sending to API:
    ```typescript
    if (password.length < 6) {
      alert('Password must be at least 6 characters');
      return;
    }
    ```
  </Accordion>
  
  <Accordion title="Provide clear error feedback">
    Display specific validation errors to users:
    - "Email already registered" vs generic "Registration failed"
    - "Username too short" vs "Invalid username"
  </Accordion>
  
  <Accordion title="Normalize user input">
    The API normalizes input, but client-side normalization improves UX:
    ```typescript
    const email = userInput.toLowerCase().trim();
    ```
  </Accordion>
</AccordionGroup>
