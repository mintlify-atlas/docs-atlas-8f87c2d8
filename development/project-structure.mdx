## Root Directory Structure

```
CUCEIUbicate/
├── App.js                    # Application entry point
├── app.json                  # Expo configuration
├── package.json              # Dependencies and scripts
├── babel.config.js           # Babel configuration
├── tsconfig.json             # TypeScript configuration
├── eas.json                  # Expo Application Services config
├── assets/                   # Global application assets
│   └── images/
│       ├── cucei.png        # App icon
│       ├── splash.png       # Splash screen
│       └── icono.png        # Android adaptive icon
├── json/                     # Static JSON data files
├── plugins/                  # Custom Expo config plugins
├── scripts/                  # Build and utility scripts
└── Src/                     # Main source code directory
    ├── Api/                 # API integration layer
    ├── auth/                # Authentication screens
    ├── Screens/             # Feature screens
    ├── components/          # Shared components
    └── assets/              # Source-level assets
```

## Src Directory - Complete Structure

The `Src/` directory contains all application source code, organized by feature and function:

### Authentication Module (`Src/auth/`)

<Accordion title="Authentication Files">
```
Src/auth/
├── LoginScreen.js           # Email/password login
├── RegisterScreen.js        # New user registration
├── CompleteProfile.js       # Post-registration profile setup
├── OnboardingScreen.js      # First-time user onboarding
└── SessionManager.js        # Session persistence utilities
```

**Key Files:**

- **`LoginScreen.js`**: Handles user authentication with email/password
- **`RegisterScreen.js`**: New account creation with validation
- **`CompleteProfile.js`**: Additional user information (career, preferences)
- **`OnboardingScreen.js`**: Swipeable introduction for first-time users
- **`SessionManager.js`**: AsyncStorage wrapper for session management
</Accordion>

### API Layer (`Src/Api/`)

<Accordion title="API Integration">
```
Src/Api/
├── login.js                 # Authentication logic
└── lib/
    └── api.js              # Supabase client and API calls
```

**Responsibilities:**
- Supabase client initialization
- User authentication and verification
- Database queries for user data
- Password hashing and validation
</Accordion>

## File Naming Conventions

The project uses several naming patterns:

<CodeGroup>
```text Screen Components
PascalCase for React components:
- LoginScreen.js
- HomePage.js
- ProfileScreen.js
```

```text Utility Files
camelCase for utilities:
- login.js
- data.js
- bottomSheetContents.js
```

```text Component Prefixes
Descriptive prefixes for related files:
- Search*.js (SearchRoute, SearchSpecific)
- Data_*.js (Data_iconos_mallas)
- Videos_*.js (Videos_data)
```
</CodeGroup>

## Module Organization Pattern

Each building module in `Src/Screens/Home/Components/BottonSheetComponent/Modulos/` follows a consistent structure:

```
Modulo_A/
├── [BuildingName].js           # Component file
└── imagenes/                   # Building-specific images
    ├── exterior.jpg
    ├── interior_1.jpg
    └── ...
```

This pattern is repeated for 50+ campus buildings (Modules A-Z, specialized buildings, entrances, etc.).

## Development Workflow

### Adding a New Screen

1. Create screen file in appropriate `Src/Screens/[Category]/` directory
2. Add navigation entry in `MyDrawer.js`
3. Import and configure in drawer navigator
4. Add drawer item with icon and label

### Adding a New Building Module

1. Create directory: `Modulos/[ModuleName]/`
2. Add component file with building information
3. Create `imagenes/` subdirectory for photos
4. Update `data.js` with new map point
5. Register in `bottomSheetContents.js`
